# Week 7 LLM Usage Report

## Scope

AI assistance was used for requirement analysis, repository/evidence inspection, documentation drafting, branch synchronization, and consistency checks. Product approval, customer confirmation, UAT execution, code review by another team member, and merge authorization remained human responsibilities.

## ChatGPT

| Area | Record |
|---|---|
| Purpose | Analyze the Assignment 6/Week 7 requirements, structure the evidence checklist, compare required claims with available repository evidence, and draft documentation language for review. |
| Files/tasks affected | Planning for the Week 7 report, retrospective, reflection, LLM report, handover wording, UAT evidence categories, PR description, and issue-update checklist. |
| Human verification | Team members checked proposed wording against issues, merged PRs, `main`, customer-confirmation information, and the private/public evidence boundary before accepting it. |
| Limitations | ChatGPT could not supply customer evidence, validate private recordings, execute UAT, approve work, or decide that a release/check/review existed without inspectable evidence. |
| Not delegated | Customer communication and confirmation, UAT execution, reviewer approval, privacy decisions, release creation, and final submission accountability. |

## Codex

| Area | Record |
|---|---|
| Purpose | Inspect PR #199 and current GitHub issues, fetch and merge current `main` without force-push, resolve documentation conflicts, implement the final Markdown artifacts, verify the public demo endpoint, and run repository consistency/link/privacy checks. |
| Files/tasks affected | `README.md`, `docs/customer-handover.md`, `docs/roadmap.md`, `docs/user-acceptance-tests.md`, and the Week 7 report, retrospective, reflection, LLM report, Sprint Review summary/transcript; PR #199 metadata and linked issue evidence. |
| Human verification | A different team member must review the PR before merge. The team remains responsible for checking factual customer statements, GitHub issue criteria, private Moodle evidence, screenshots, slides, and final release readiness. |
| Limitations | Codex cannot turn team verification into customer execution, infer private recording content that was not provided, fabricate screenshots or links, or bypass GitHub branch protection/review requirements. |
| Not delegated | Customer acceptance itself, customer-side deployment, customer-executed UAT, product approval, independent review, release/tag creation, private evidence preparation, and Moodle submission. |

## Evidence controls

- AI did **not** provide customer confirmation.
- AI did **not** conduct UAT or customer-side deployment.
- AI did **not** approve the product or replace the required human reviewer.
- AI-generated text was checked against repository files, merged PRs, current issues, and the confirmed status supplied by the team.
- No fabricated links, tests, reviews, screenshots, releases, recordings, or customer actions were used.
- Where evidence was incomplete, the documentation uses `Discussed`, `Team-verified`, `Not executed`, `Pending`, or an explicit limitation rather than upgrading the claim.
